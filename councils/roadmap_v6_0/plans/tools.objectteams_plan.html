Introduction

Tentative plan for the next Object Teams releases.

Next to the 0.7.x stream, we hope to graduate early in 2011 to yield a 2.0.0 release. Meanwhile the 0.8.0 stream participates in the Indigo simultaneous release train.

Release Deliverables

Themes and Priorities

Synchronize with the Simultaneous Release train

Milestones towards the OTDT release 0.8.0 participate in the Indigo simultaneous release train.

API Consolidation

Final changes to the package structure and API will be made within the 0.8 stream in preparation for the graduation.

Improve support for generics

Type parameters in roles and teams intricately interact with dependent types in OT/J. During the 0.8 stream more tests for the various applications of type parameters within roles and teams will be added and the implementation will be improved accordingly.
  • Committed
    • [compiler][generics] CCE in generic nested team [325397] (target milestone: 0.8 M4)
    • [compiler][generics] static role method cannot access type variable of the enclosing team [331869] (target milestone: 0.8 M4)
    • [compiler][generics] implicit inheritance must apply substitution for type variables from outer scope [331877] (target milestone: 0.8 M4)
    • [compiler][generics] import issues with inherited playedBy binding inside parameterized team [332795] (target milestone: 0.8 M5)
    • [compiler][generics] instantiating a generic role cannot be type-checked [332801] (target milestone: 0.8 M5)

Bugfixes and Enhancements

Regular inbox tracking and bug fixing.
  • Committed
    • No items.

Improve Compatibility

  1. Some of the techniques applied in the OTDT implementation potentially interact with other plug-ins (JDT/Core patch, bytecode weaving). It will be tested which plug-ins might be affected and if possible undesirable interactions will be resolved.
  2. A prototype of an alternative bytecode weaver exists that is designed to improve the user experience in two ways:
    • Better support Equinox lazy loading of bundles in order to reduce start-up time.
    • Support runtime weaving of new OT/Equinox bundles even without restarting the Equinox framework.
Improvements in this category will be released time permitting.
